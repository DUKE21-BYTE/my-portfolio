How I Built My First Responsive Website

January 2026 3 min read

When I first started coding, I thought making a website was just about putting boxes on a screen. I positioned everything perfectly on my laptop, felt like a genius, and then opened it on my phone. Disaster. Text was overflowing, images were huge, and I had to scroll horizontally just to read a sentence.

The "Pixels are Fixed" Trap

My biggest mistake was using fixed units like `px` for everything. I would write:

.container { width: 1200px; /* Big mistake! */ margin: 0 auto; }

This forced the browser to trying to fit 1200 pixels onto a phone screen that was only 375 pixels wide. The fix was shifting my mindset to relative units. Instead of pixels, I learned to love percentages (`%`), viewport units (`vw/vh`), and `rem`.

Discovering Flexbox

Before Flexbox, I was trying to use `float: left` to stack things next to each other. It was a nightmare of clearing floats and broken layouts. Flexbox changed everything. Suddenly, centering a `div` wasn't rocket science anymore:

.hero { display: flex; justify-content: center; align-items: center; /* The holy grail */ }

The Magic of Media Queries

The final piece of the puzzle was media queries. This is where you tell CSS, "Hey, if the screen is smaller than a tablet, do this instead."

@media (max-width: 768px) { .nav-menu { display: none; /* Hide regular menu on mobile */ } .grid { grid-template-columns: 1fr; /* Stack into one column */ } }

Conclusion

Building responsive sites isn't just about code; it's about empathy for the user. Whether they are on a massive desktop or a budget smartphone, the experience should be seamless. That's the standard I now hold for every project I build, including this portfolio.
